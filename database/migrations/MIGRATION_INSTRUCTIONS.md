# Инструкция по применению миграции конфигураций

## Шаг 1: Применить миграцию базы данных

Выполните миграцию одним из следующих способов:

### Вариант A: Через pgAdmin
1. Откройте pgAdmin
2. Подключитесь к базе данных `accounting_1c`
3. Откройте Query Tool (F5)
4. Скопируйте содержимое файла `database/migrations/001-configurations.sql`
5. Вставьте в Query Tool и выполните (F5)

### Вариант B: Через командную строку (если PostgreSQL в PATH)
```bash
psql -U postgres -d accounting_1c -f "database/migrations/001-configurations.sql"
```

### Вариант C: Через Node.js скрипт
```bash
cd server
node src/scripts/run-migration.js
```

## Шаг 2: Проверить применение миграции

В pgAdmin или psql выполните:
```sql
SELECT COUNT(*) FROM configurations;
```

Должно вернуться 20 (количество конфигураций 1С).

## Шаг 3: Перезапустить серверы

После применения миграции перезапустите backend и frontend серверы:

```bash
# Backend (если был запущен)
cd server
npm run dev

# Frontend (если был запущен)
cd client-accounting
npm run dev
```

## Шаг 4: Проверить функционал

1. Откройте http://localhost:3000
2. Войдите в систему (admin / admin123)
3. Если конфигурация не выбрана, вы будете автоматически перенаправлены на страницу выбора
4. Выберите конфигурацию (например, "1С:Бухгалтерия предприятия (ред. 3.0)")
5. Убедитесь, что меню приложения обновилось и отображает только модули выбранной конфигурации
6. Перейдите в Настройки (последний пункт меню) для смены конфигурации

## Возможные проблемы

**Проблема**: "psql не распознано"
**Решение**: Используйте pgAdmin (Вариант A) или Node.js скрипт (Вариант C)

**Проблема**: "Table already exists"
**Решение**: Миграция уже применена, можно продолжать работу

**Проблема**: Меню не обновляется
**Решение**: 
1. Очистите кеш браузера (Ctrl+Shift+Delete)
2. Перезагрузите страницу (Ctrl+F5)
3. Проверьте консоль браузера на наличие ошибок
