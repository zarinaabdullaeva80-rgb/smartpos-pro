## Настройка 1С {#1c-setup}

### 1. Создать HTTP-сервис в 1С

1. Открыть конфигурацию в режиме Конфигуратор
2. Добавить новый HTTP-сервис: `exchange`
3. Создать URL шаблон: `/exchange`
4. Добавить методы:
   - `GET products` - получить товары
   - `POST products` - принять товары
   - `GET sales` - получить продажи
   - `POST sales` - принять продажи

### 2. Включить Basic Authentication

Конфигуратор → Свойства конфигурации → Интернет-поддержка:
- ✅ Использовать HTTP-сервисы
- ✅ Аутентификация: Basic Auth

### 3. Публикация на веб-сервере

После компиляции и публикации сервис будет доступен:
```
http://your-server/accounting/hs/exchange
```

### Регламентное задание для автоматической синхронизации

```1c

    // Подготовить данные

    // Сериализация

    ЗаписьJSON.УстановитьСтроку();
    ЗаписатьJSON(ЗаписьJSON, Данные);
    ТелоЗапроса = ЗаписьJSON.Закрыть();
    
    // HTTP запрос

    Иначе

            "Ошибка синхронизации: " + Ответ.КодСостояния + " " + Ответ.ПолучитьТелоКакСтроку());
```

## Troubleshooting {#troubleshooting}

### Ошибка: "Connection refused"

**Причина:** 1С сервер недоступен

**Решение:**
1. Проверить, что IIS/Apache запущен
2. Проверить публикацию базы на веб-сервере
3. Проверить firewall

### Ошибка: "401 Unauthorized"

**Причина:** Неверные учётные данные

**Решение:**
1. Проверить логин/пароль в `sync_settings`
2. Проверить, что пользователь имеет права на HTTP-сервисы в 1С

### Товары дублируются

**Причина:** Не работает маппинг external_id

**Решение:**
1. Проверить заполнение `external_id` в JSON
2. Проверить таблицу `external_id_mapping`
3. Использовать уникальные поля (штрихкод, артикул) для поиска

### Медленная синхронизация

**Решение:**
1. Использовать `date_from` параметр
2. Ограничить `limit`
3. Использовать асинхронную очередь `sync_queue`
4. Включить индексы в 1С на поля поиска
