# Быстрый старт для сборки .exe файлов

## Шаг 1: Установка Node.js

1. **Скачайте Node.js:**
   - Перейдите на [https://nodejs.org/](https://nodejs.org/)
   - Скачайте версию **LTS (Long Term Support)** для Windows
   - Текущая LTS версия: v20.x.x

2. **Установите Node.js:**
   - Запустите скачанный установщик
   - Следуйте инструкциям (используйте настройки по умолчанию)
   - **ВАЖНО**: Отметьте галочку "Automatically install necessary tools"
   - После установки **перезапустите компьютер**

3. **Проверьте установку:**
   - Запустите файл `check-nodejs.bat`
   - Вы должны увидеть версии Node.js и npm

## Шаг 2: Сборка приложений

### Вариант A: Собрать оба приложения

Просто запустите:
```
build-all.bat
```
Это займет 10-15 минут.

### Вариант B: Собрать по отдельности

**1С Бухгалтерия:**
```
build-1c-accounting.bat
```

**Админ панель:**
```
build-admin-panel.bat
```

## Шаг 3: Установка приложений

После завершения сборки:

1. Откроются папки с установщиками
2. Найдите файлы:
   - `1С Бухгалтерия Setup X.X.X.exe`
   - `1С Админ Панель Setup X.X.X.exe`
3. Запустите их для установки

## Альтернатива: Запуск без сборки exe

Если не хотите собирать exe, можно запустить как web приложения:

### 1. Запустить backend сервер
```bash
cd server
npm install
npm start
```

### 2. Запустить 1С Бухгалтерия (в новом окне PowerShell)
```bash
cd client-accounting
npm install
npm run dev
```
Откройте http://localhost:3000

### 3. Запустить Админ панель (в новом окне PowerShell)
```bash
cd client-admin
npm install
npm run dev
```
Откройте http://localhost:3001

## Решение проблем

### "npm не является внутренней или внешней командой"
- Node.js не установлен или не добавлен в PATH
- Перезапустите PowerShell после установки Node.js
- Перезагрузите компьютер

### Ошибка при установке зависимостей
- Убедитесь, что есть подключение к интернету
- Попробуйте запустить PowerShell от имени администратора

### Долгая установка зависимостей
- Это нормально, особенно electron (~300MB)
- Первая установка может занять 5-10 минут

### Ошибка при сборке
- Убедитесь, что установлены Visual Studio Build Tools
- Запустите установщик Node.js и выберите "Tools for Native Modules"

## Что делать после сборки

2. ✅ Запустите backend сервер (см. README.md)
3. ✅ Откройте "1С Бухгалтерия" и "1С Админ Панель"
4. ✅ Войдите с логином `admin` и паролем `admin123`
