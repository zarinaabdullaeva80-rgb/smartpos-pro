## Возникшие проблемы и решения

### ❌ Проблема с подписанием кода (winCodeSign)
**Ошибка**: `Cannot create symbolic link : Отказано в требуемых правах`

**Причина**: 
- Отсутствие цифрового сертификата для подписи кода
- Проблемы с правами доступа при создании символических ссылок

**Решение**:
- Изменён тип сборки с `nsis` (установщик) на `dir` (portable версия)
- Portable версия не требует подписи и работает без установки
- EXE файл успешно создан несмотря на ошибки подписания

## Как запустить приложение

### Способ 1: Прямой запуск
1. Откройте папку `Desktop/1С Бухгалтерия - Desktop App/`
2. Дважды кликните на `1С Бухгалтерия.exe`

### Способ 2: Через bat файл
1. Откройте папку `Desktop/1С Бухгалтерия - Desktop App/`
2. Дважды кликните на `ЗАПУСК.bat`

### ⚠️ Важно! Перед запуском:
1. **Backend сервер должен быть запущен** на `http://localhost:5000`
   ```bash
   cd server
   npm run dev
   ```

2. **PostgreSQL должен быть запущен**

## Распространение приложения

### Для использования на другом компьютере:
1. Скопируйте всю папку `1С Бухгалтерия - Desktop App` на флешку
2. Перенесите на целевой компьютер
3. Убедитесь что backend сервер доступен
4. Запустите `1С Бухгалтерия.exe`

### Для создания установщика с автоустановкой:
1. Получите сертификат для подписи кода
2. В `package.json` измените `"target": "dir"` на `"target": "nsis"`
3. Настройте сертификат в переменных окружения
4. Запустите `npm run electron:build`
