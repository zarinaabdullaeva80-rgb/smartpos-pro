# Инструкции по запуску и тестированию систем

## 🚀 Быстрый старт

### 1. Убедитесь что БД инициализирована:
```bash
cd database
npm run quick-init
```

### 2. Запустите тест всех систем:
```bash
.\TEST_ALL_SYSTEMS.bat
```

### 3. Запустите системы в отдельных терминалах:

**Terminal 1 - Backend Server:**
```bash
cd server
npm start
```
Будет доступен на: `http://localhost:5000`

**Terminal 2 - Desktop приложение:**
```bash
cd client-accounting
npm start
```
Будет доступен на: `http://localhost:3000`

**Terminal 3 - Admin Panel:**
```bash
cd client-admin
npm start
```
Будет доступен на: `http://localhost:3001`

**Terminal 4 - Mobile POS:**
```bash
cd mobile-pos
npx expo start
```
Сканируйте QR-код в Expo Go приложении

## 📱 Настройка мобильного доступа

### Вариант 1: Локальная сеть (WiFi)

1. **Узнайте IP адрес вашего компьютера:**
   ```bash
   ipconfig
   ```
   Найдите IPv4 адрес (например: `192.168.1.100`)

2. **Обновите API URL в mobile-pos:**
   Файл: `mobile-pos/src/services/api.js`
   ```javascript
   const API_URL = 'http://192.168.1.100:5000';
   ```

3. **Подключите телефон к той же WiFi сети**

4. **Запустите Expo:**
   ```bash
   cd mobile-pos
   npx expo start
   ```

5. **Сканируйте QR-код в Expo Go**

### Вариант 2: Через интернет (Ngrok)

1. **Установите ngrok:**
   ```bash
   npm install -g ngrok
   ```

2. **Запустите ngrok:**
   ```bash
   ngrok http 5000
   ```

3. **Скопируйте URL** (например: `https://abc123.ngrok.io`)

4. **Обновите API URL во всех клиентах:**
   - `mobile-pos/src/services/api.js`
   - `client-accounting/.env`: `REACT_APP_API_URL=https://abc123.ngrok.io`
   - `client-admin/.env`: `REACT_APP_API_URL=https://abc123.ngrok.io`

## 🐛 Возможные проблемы

### Проблема: "Cannot connect to server"
**Решение:** Проверьте что backend запущен на порту 5000

### Проблема: "Database connection failed"
**Решение:** 
```bash
cd database
npm run quick-init
```

### Проблема: "Mobile can't connect"
**Решение:** Проверьте что телефон и компьютер в одной WiFi сети

### Проблема: "Login failed"
**Решение:** Проверьте `database/init-users.json` и переинициализируйте БД
