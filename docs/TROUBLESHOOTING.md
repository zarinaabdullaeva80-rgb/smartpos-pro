# Руководство по устранению неполадок

## Проблемы с аутентификацией

### Ошибка 401 "Недействительный токен" после успешного входа

**Симптомы:**
- Login API возвращает 200 OK с токеном
- Последующие запросы к защищённым endpoints возвращают 401
- Frontend автоматически выполняет logout

**Причина:** Несоответствие схемы базы данных в SQL запросах middleware

**Решение:** Проверьте файл `server/src/middleware/auth.js`
- Убедитесь, что SQL запрос использует существующие столбцы (`role`, `user_level`)
- Проверьте структуру `req.user` на соответствие данным из БД

**Подробнее:** См. `AUTH_FIX_DOCUMENTATION.md`

---

### JWT_SECRET не определён

**Симптомы:**
- Ошибки при проверке токенов
- Логи показывают "Using JWT_SECRET: ENV_VAR" вместо "FALLBACK_KEY"

**Решение:**
1. Создайте файл `.env` в директории `server/`
2. Добавьте строку: `JWT_SECRET=your-secure-secret-key-here`
3. Перезапустите сервер

**Fallback:** Если `.env` не создан, используется `'your-secret-key-change-in-production'`

## Проблемы с базой данных

### PostgreSQL не запущен

**Симптомы:**
- Ошибки подключения к БД при запуске сервера
- `connection refused` ошибки

**Решение:**
```powershell
# Проверка статуса PostgreSQL
sc query postgresql-x64-14

# Запуск службы
net start postgresql-x64-14
```

---

### Несоответствие схемы и фактической структуры БД

**Симптомы:**
- SQL ошибки "столбец не существует"
- Несоответствие между `schema.sql` и фактической БД

**Диагностика:**
Проверьте структуру таблицы через pgAdmin или SQL:
```sql
SELECT column_name, data_type 
FROM information_schema.columns 
WHERE table_name = 'users';
```

**Решение:** Используйте структуру, соответствующую фактической БД

## Проблемы с Redis

### Redis connection refused

**Симптомы:**
- Частые ошибки `ECONNREFUSED ::1:6379` в логах сервера
- Приложение работает, но логи засорены ошибками

**Причина:** Redis не установлен или не запущен

**Решение:**
1. **Опция 1:** Установить и запустить Redis for Windows
2. **Опция 2:** Отключить Redis в коде (если не используется для кэширования)

## Проблемы с портами

### Порт уже используется

**Симптомы:**
- `Error: listen EADDRINUSE :::5000`
- Сервер не может запуститься

**Решение:**
```powershell
# Найти процесс, использующий порт 5000
netstat -ano | findstr :5000

# Завершить процесс (замените PID на фактический)
taskkill /PID <PID> /F
```
