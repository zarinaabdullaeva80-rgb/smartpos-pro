# Google Sheets Integration Guide

Подробное руководство по настройке синхронизации данных с Google Sheets.

## Обзор

Система автоматически синхронизирует данные с Google Sheets каждые 5 минут:
- **Товары** - вся номенклатура с ценами и остатками
- **Продажи** - все проведенные документы продаж
- **Остатки** - текущие остатки на складах
- **Статистика** - сводная аналитика (выручка, топ товары, прибыль)

## Настройка Google Cloud

### Шаг 1: Создание Проекта

1. Откройте [Google Cloud Console](https://console.cloud.google.com/)
2. Нажмите **Select a project** → **New Project**
3. Введите название: `1C-Accounting`
4. Нажмите **Create**

### Шаг 2: Включение APIs

1. В меню слева выберите **APIs & Services** → **Library**
2. Найдите и включите:
   - **Google Sheets API**
   - **Google Drive API**

### Шаг 3: Создание Service Account

1. Перейдите в **APIs & Services** → **Credentials**
2. Нажмите **Create Credentials** → **Service Account**
3. Заполните:
   - **Service account name:** `1c-sync`
   - **Service account ID:** `1c-sync` (автоматически)
   - **Description:** `Синхронизация данных 1С`
4. Нажмите **Create and Continue**
5. Пропустите опциональные шаги (**Done**)

### Шаг 4: Создание JSON Ключа

1. На странице **Service Accounts** найдите созданный аккаунт
2. Нажмите на него → вкладка **Keys**
3. **Add Key** → **Create new key**
4. Выберите **JSON**
5. Ключ автоматически скачается
6. Переименуйте в `google-credentials.json`
7. Скопируйте email Service Account (например, `1c-sync@project-id.iam.gserviceaccount.com`)

### Предоставление Доступа

1. Откройте созданную таблицу
2. Нажмите **Share** (Поделиться)
3. Вставьте email Service Account
4. Установите права: **Editor** (Редактор)
5. Снимите галочку **Notify people**
6. Нажмите **Share**

## Настройка Backend

### 1. Копирование Credentials

Скопируйте `google-credentials.json` в папку `server`:

```bash
# Windows
copy C:\Downloads\google-credentials.json "C:\Users\user\Desktop\1С бухгалтерия\server\"
```

### 2. Обновление .env

Добавьте в `server/.env`:
```env
GOOGLE_SHEETS_CREDENTIALS_PATH=./google-credentials.json
GOOGLE_SHEETS_SPREADSHEET_ID=ваш_spreadsheet_id
```

### 3. Проверка Кода Синхронизации

Код синхронизации уже реализован в [server/src/services/googleSheets.js](file:///c:/Users/user/Desktop/1С бухгалтерия/server/src/services/googleSheets.js).

**Основные функции:**
- `initGoogleSheets()` - инициализация API
- `syncAllData()` - синхронизация всех данных
- Автоматическая синхронизация каждые 5 минут

### 4. Перезапуск Сервера

```bash
cd server
npm run dev
```

После запуска в логах должно появиться:
```
✓ Google Sheets API инициализирован
✓ Синхронизация с Google Sheets выполнена успешно
```

## Тестирование

### Ручная Проверка

1. Создайте несколько товаров в системе
2. Создайте несколько продаж
3. Подождите до 5 минут
4. Откройте Google Таблицу
5. Проверьте, что данные появились

### Ручная Синхронизация

Можно добавить endpoint для ручной синхронизации:

```javascript
// В server/src/index.js добавить:
app.post('/api/sync-sheets', async (req, res) => {
    try {
        await syncAllData();
        res.json({ message: 'Синхронизация выполнена успешно' });
    } catch (error) {
        res.status(500).json({ error: error.message });
    }
});
```

Затем вызвать:
```bash
curl -X POST http://localhost:5000/api/sync-sheets
# или
curl -X POST https://your-app.onrender.com/api/sync-sheets
```

## Решение Проблем

### Ошибка "Could not load credentials"

**Причина:** Файл `google-credentials.json` не найден или путь неверный.

**Решение:**
```bash
# Проверить наличие файла
ls server/google-credentials.json

# Проверить путь в .env
cat server/.env | grep GOOGLE_SHEETS
```

### Ошибка "The caller does not have permission"

**Причина:** Service Account не имеет доступа к таблице.

**Решение:**
1. Откройте Google Таблицу
2. Проверьте, что email Service Account добавлен с правами Editor

### Ошибка "Spreadsheet not found"

**Причина:** Неверный Spreadsheet ID.

**Решение:**
1. Откройте таблицу в браузере
2. Скопируйте ID из URL
3. Обновите `.env`

### Данные не синхронизируются

**Проверка:**
```bash
# Посмотреть логи сервера
# Должно быть:
✓ Google Sheets API инициализирован
# При синхронизации:
Синхронизация товаров: 25 записей
Синхронизация продаж: 10 записей
```

**Если в логах ошибка:**
- Проверьте credentials
- Проверьте доступ к таблице
- Проверьте Spreadsheet ID

## Польза Google Sheets

### Автоматическое Резервное Копирование
- Все данные дублируются в Google Sheets
- Можно восстановить данные при сбое БД

### Аналитика и Отчеты
- Используйте формулы Google Sheets для анализа
- Создавайте графики и сводные таблицы
- Пример формулы для общей выручки: `=SUM(Продажи!G:G)`

### Доступ для Руководства
- Предоставьте доступ только на чтение
- Руководство может видеть актуальные данные
- Не нужно давать доступ к админ панели

### Интеграция с Другими Сервисами
- Google Data Studio для дашбордов
- Google Apps Script для автоматизации
- Экспорт в Excel для дополнительного анализа
